Provides support for Attachment handling and storing.

AbstractAttachment extends the Attachment interface (see 'org.ws4d.java.attachment'). Provides access to the metadata (content ID, content type, transfer encoding) and raw data of an attachment. Is a container for Attachment properties. DefaultAttachmentStore extends AttachmentStore and defines the common ground for storing attachments. Specific types of storage are handled by extending classes.

Attachments can be stored in three different ways:

  • within memory (as a byte buffer), see the MemoryAttachment class
  • within external storage (like the file system), see the FileAttachment class
  • as a byte stream, see the classes InputStreamAttachment and OutputStreamAttachment
